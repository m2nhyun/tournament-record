# Tournament Record

아마추어 테니스 모임/클럽의 경기 기록을 쉽고 신뢰 가능하게 남기는 서비스.

## Product

- 클럽 생성 및 초대 코드 기반 멤버 운영
- 초대 링크 기반 게스트 참가
- 단식/복식 경기 기록
- 테니스 게임 단위 점수 입력(목표 게임 선택, 미완료 게임 저장 가능)
- 상대 확인 기반 경기 확정/거절 플로우
- 경기 히스토리 조회(카드/리스트, 필터, 무한 스크롤)
- 클럽 리더보드(확정 경기 기준 승/패/경기 수)

## Core UX Principles

1. 기록은 빠르고 직관적이어야 한다.
2. 용어는 테니스 실사용 관점(게임 중심)으로 통일한다.
3. 히스토리는 스캔 중심으로 설계한다.
4. 승/패 표시는 로그인 사용자 기준으로 정확해야 한다.

## Match History

- `카드 모드`: 경기 맥락(유형/상태/날짜/게임 스코어) 확인
- `리스트 모드`: `팀A x:y 팀B` 최소 정보만 노출
- 미확정 경기(`submitted`, `disputed`)는 중립 표현 우선
- 필터: 날짜 + 상대 이름 (접기/펼치기)
- 무한 스크롤: 점진 로딩

## Design Direction

- 담백한 UI, 높은 가독성
- 배지 남용보다 정보 밀도와 스캔 속도 우선
- 승/패 시각 강조는 연한 그린/레드 톤 사용

## Tech

- Next.js (App Router) + TypeScript
- Tailwind CSS + shadcn/ui
- Supabase (Postgres/Auth/RLS)
- Vercel

## Auth

- Kakao OAuth 로그인 지원
- Email/Password 로그인 지원
- 초대 링크 진입 게스트 참가 지원
