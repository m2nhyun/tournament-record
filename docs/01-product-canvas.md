# Product Canvas

## 제품 한 줄 정의

아마추어 테니스 클럽 운영진이 정모/번개 경기 기록을 현장에서 남기고, 멤버가 내 전적과 클럽 활동을 확인하는 클럽 레코드 플랫폼.

## 핵심 도메인: 클럽 레코드

이 제품의 실질적인 핵심은 **클럽 레코드(Club Record)** 기능이다. 클럽 레코드는 다음 흐름으로 동작한다.

```
이벤트 생성 → 참가자 등록 → 슬롯/자동 편성 → 경기 진행 → 결과 입력 → 확정
                                                                 ↓
                                                     히스토리 / 월간 카드 / 랭킹
```

### 이벤트 (Club Record Event)

- 클럽 운영진이 정모 일정을 이벤트로 등록한다.
- 이벤트는 날짜, 시작/종료 시각, 코트 수로 구성된다.
- 상태: `draft` / `open` / `in_progress` / `completed` / `cancelled`
- 종료 시각이 지나거나 `completed`/`cancelled`이면 읽기 전용으로 전환된다.

### 슬롯 (Club Record Event Slot)

- 이벤트에서 코트 × 시간대 단위의 경기 자리다.
- 슬롯에는 최대 1개의 경기가 연결된다.
- 상태: `scheduled` / `ready` / `completed` / `cancelled`
- 확정된 경기가 연결된 슬롯은 삭제/수정이 차단된다.

### 참가자 (Club Record Event Participant)

- 정회원(`member`)과 게스트(`guest`) 두 유형이 있다.
- 참가자에게 도착 시각을 설정하면, 자동 편성 시 해당 슬롯 이후부터만 배치된다.
- 게스트는 초대 코드 링크를 통해 이벤트에 진입한다.

### 자동 편성 (Auto Assignment)

- 운영진이 자동 편성을 실행하면, 가용 참가자와 열린 슬롯을 매핑해 경기를 생성한다.
- 편성 변경 사항이 있거나 열린 슬롯이 남아 있으면 재실행을 권장한다.
- 자동 편성 결과는 수동으로 덮어쓸 수 있다.

### 경기 (Club Record Match)

- 슬롯에 할당된 경기. 팀 1 vs 팀 2 구성.
- `assignment_mode`: `auto` | `manual`
- 상태: `pending_result` → `confirmed` / `cancelled`
- 확정된 경기는 직접 삭제가 차단된다.

### 결과 입력

- 클럽 레코드 경기는 참가자(정회원)가 결과를 입력/제출하면 `pending_result → confirmed`로 전환된다(별도 상대 확인 단계 없음).
- 운영진은 어느 경기든 결과를 사후 입력하거나 수정할 수 있다.
- 일반 matches 경로(`/clubs/[clubId]/matches/*`)는 별도 `submitted → confirmed/disputed` 흐름을 유지하며, 클럽 레코드 매치와는 상태 머신이 다르다.

### 랭킹 / 클럽 레코드 멤버

- 클럽 레코드 전용 랭킹 테이블(`club_record_members`)이 별도로 관리된다.
- `sync_club_record_members` RPC로 클럽 정회원과 동기화한다.
- 운영진은 랭킹 순서를 수동으로 조정할 수 있다.

### 히스토리 & 월간 카드

- 멤버는 자신이 참가한 확정 경기의 전체 이력을 조회한다.
- 월간 카드: 클럽 내 월별 승/패/무/출전 수와 승률을 퍼블릭으로 공개한다.

## 사용자 유형

### 1) 클럽 운영진 (owner / manager)

- 이벤트 생성, 참가자 등록, 자동 편성 실행, 결과 입력/수정
- 클럽 레코드 멤버 동기화, 랭킹 조정

### 2) 활동 멤버 (member)

- 참가 이벤트 확인, 결과 입력 (본인 경기만), 내 히스토리 조회
- 월간 카드 / 랭킹 조회

### 3) 게스트

- 초대 코드로 이벤트 진입, 경기 참가
- 결과 입력 불가, 히스토리/랭킹 조회 불가

---

## 활성화 루프

### 정모 기록 루프

1. 운영진이 이벤트 생성 → 참가자 등록 → 자동 편성 실행
2. 경기 진행 → 참가 멤버가 결과 입력 → 자동 확정
3. 확정 결과가 히스토리 / 월간 카드 / 랭킹에 반영
4. 멤버가 내 기록 확인을 위해 재방문

### 클럽 경쟁 루프

- 월간 카드에서 이번 달 승률과 경기 수를 비교
- 랭킹 변화가 다음 정모 참여 동기로 이어짐

### 보조 트랙: 단일 매치 / 일정 모집 (matches feature)

`club_record`와 별개로 `/clubs/[clubId]/matches/*`, `/clubs/[clubId]/history`, `/clubs/[clubId]/schedules/*` 라우트가 다음 용도로 살아 있다.

- 일회성 경기 기록 (`/matches/new`)
- 일정 잡기 진입 (`/matches/new?mode=schedule` → 일정 모집)
- 경기 상세 보기 / 확인 처리 (`/matches/[matchId]`)
- 일반 경기 히스토리 뷰 (`/history`)

바텀 네비게이션에서는 노출되지 않고, club_record 대시보드의 confirmation 카드와 일정 카드의 내부 링크에서 진입한다. `/leaderboard`는 현재 어떤 Link에서도 참조되지 않는 orphan 라우트이며 정책 결정 대기 상태다.

## 비범위 (현재)

- 글로벌 레이팅 / 외부 클럽 간 교류전
- 경기 영상 분석
- 범용 커뮤니티/게시판

> 인증은 현재 카카오 OAuth와 이메일/비밀번호(`signInWithPassword` / `signUp`)를 모두 활성으로 유지한다. 자세한 흐름은 `docs/11-auth-onboarding-design.md` 참고.

## 핵심 지표

- 이벤트 생성 후 첫 경기 결과 입력까지 걸리는 시간
- `pending_result → confirmed` 전환율
- 월간 카드 조회 수 (재방문 지표)
- 이벤트당 평균 확정 경기 수
- 클럽별 주간 활성 이벤트 수
