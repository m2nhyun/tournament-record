# Product Canvas

## 제품 한 줄 정의

아마추어 테니스 모임/클럽의 경기 기록을 쉽고 신뢰 가능하게 남기는 서비스.

## 제품 포지셔닝

- 이 서비스의 1차 고객은 개인 선수가 아니라 `클럽 운영진`이다.
- 사용자는 멤버 전체이지만, 도입과 정착은 운영진의 편의성에서 시작된다.
- 따라서 제품은 `개인 기록 앱`보다 `클럽 운영용 경기 기록 플랫폼`으로 정의한다.

## 우리가 먼저 풀어야 하는 일

- 운영진이 정모/번개 결과를 엑셀이나 카톡 대신 빠르게 남길 수 있어야 한다.
- 멤버는 내 경기, 내 전적, 클럽 내 최근 활동을 보려고 다시 들어와야 한다.
- 기록이 쌓일수록 클럽 히스토리와 리더보드의 신뢰가 커져야 한다.

## 문제

- 경기 데이터가 엑셀, 카톡, 기억에 분산된다.
- 기록 누락/오입력으로 운영 신뢰가 떨어진다.
- 히스토리가 없어 공정한 운영과 다음 확장이 어렵다.
- 기록 직후 상대 확인 절차가 없으면 결과 신뢰와 분쟁 대응이 약하다.

## 가설

1. 기록 루프를 짧게 만들면 기록률이 올라간다.
2. 클럽 단위 운영 도구를 제공하면 소수 사용자에서도 도입된다.
3. 기록/수정 이력이 있으면 분쟁 비용이 줄어든다.
4. 한국 사용자군에서는 카카오 로그인 기반 가입/복귀가 이메일 OTP보다 초기 진입 장벽이 낮다.
5. 상대 팀 확인 기반 확정 절차가 있으면 기록 신뢰도와 리더보드 수용도가 올라간다.
6. 운영진이 "정모 끝나고 30초 안에 기록"할 수 있으면 클럽 내 반복 사용이 생긴다.
7. 멤버는 전체 통계보다 `내 경기`, `내 상대`, `클럽 내 비교`를 더 자주 확인한다.
8. 초대 링크와 게스트 진입이 부드러우면 클럽 단위 배포 비용이 낮아진다.

## 핵심 사용자 유형

### 1) 클럽 운영진

- 목표: 경기 기록 누락을 줄이고, 클럽 활동 히스토리를 남긴다.
- 성공 조건: 초대, 기록, 확인 요청까지 현장에서 빠르게 끝난다.

### 2) 활동 멤버

- 목표: 내 전적과 최근 경기, 클럽 내 순위를 확인한다.
- 성공 조건: 경기 후 바로 기록이 보이고, 확인 요청 처리와 조회가 쉽다.

### 3) 게스트/신규 참가자

- 목표: 복잡한 가입 없이 클럽 흐름에 합류한다.
- 성공 조건: 초대 링크 진입, 닉네임 입력, 경기 조회/참가가 매끄럽다.

## 활성화 루프

### 1) 정모 기록 루프

- 운영진이 경기 종료 직후 결과를 저장한다.
- 상대 팀이 승인/거절한다.
- 확정 결과가 히스토리와 리더보드에 반영된다.
- 멤버는 내 기록 확인을 위해 다시 들어온다.

### 2) 클럽 내 경쟁 루프

- 멤버는 내 최근 경기와 순위 변화를 확인한다.
- 특정 상대/파트너와의 전적이 대화 소재가 된다.
- 활동량과 성적이 보이면 다음 경기 참여 동기가 생긴다.

### 3) 운영 편의 루프

- 운영진은 단톡/엑셀 정리 대신 서비스에서 기록을 남긴다.
- 신규 멤버는 누적 히스토리를 통해 클럽 분위기와 활동량을 이해한다.
- 기록 자산이 쌓일수록 서비스 이탈 비용이 커진다.

## 1차 MVP 범위 (기록 단계)

- 카카오 로그인 기반 사용자 식별/세션
- 모임/클럽 생성 및 참가 코드/초대 링크
- 게스트 참가(권한 제한)
- 경기 생성(단식/복식), 점수 입력, 미완료 게임 저장
- 상대 확인 기반 경기 확정/거절
- 개인/모임 경기 히스토리
- 기본 리더보드(확정 경기 기준 승/패, 경기 수)
- 수정 이력

## MVP 원칙

1. 운영진이 쓰기 불편하면 클럽 단위 도입은 일어나지 않는다.
2. 입력보다 조회가 재방문을 만든다. 홈은 클럽 대시보드여야 한다.
3. 입력 시간 단축이 기능 수보다 우선이다. 타이핑보다 선택형을 유지한다.
4. 데이터 신뢰가 리더보드 신뢰를 만든다. 확정 전 결과는 과장하지 않는다.
5. 게스트 허용은 유지하되, 권한은 보수적으로 제한한다.

## MVP 우선순위

### P0. 클럽 도입과 첫 기록

- 클럽 생성
- 초대 링크 진입
- 게스트/정회원 참가
- 첫 경기 기록

### P1. 정모 후 기록 완료

- 경기 저장
- 상대 확인 요청
- 확정/재검토 처리
- 히스토리 반영

### P2. 재방문 동기

- 내 최근 경기
- 클럽 최근 경기
- 리더보드
- 미확정 경기 확인

### P3. 운영 자동화

- 정모 기록 템플릿
- 월간 활동 요약
- 활동 멤버/비활성 멤버 가시화

## 핵심 지표

- 경기 후 24시간 내 기록률
- `submitted -> confirmed` 전환율
- 확인 대기 평균 소요 시간
- 주간 활성 모임 수
- 사용자당 주간 평균 기록 수
- 기록 수정/분쟁 발생률
- 4주차 잔존 모임 비율
- 클럽 생성 후 첫 경기 기록까지 걸리는 시간
- 초대 링크 진입 대비 실제 참가 완료율
- 클럽별 주간 경기 기록 운영진 수

## 비범위

- 글로벌 레이팅
- 교류전 자동 매칭
- 영상 분석
- 범용 커뮤니티/게시판 기능
