# Design Handoff (Replit / External)

디자인 시안을 외부에서 만든 뒤 이 프로젝트에 반영할 때 필요한 최소 명세.

## 목표 톤

- 담백함: 과한 장식/애니메이션 최소
- 사용성 우선: 기록 입력 속도, 실수 방지
- 모바일 우선: iPhone SE/mini급에서도 정보 구분 명확

## 화면 우선순위

1. 홈(클럽 생성/참가/내 클럽)
2. 경기 생성
3. 점수 입력
4. 히스토리 목록

## 반응형 기준

- 최소 기준 폭: 320px
- 버튼 최소 높이: 44px 이상
- 본문 기본 폰트: 14~16px
- 1열 우선 -> `sm` 이상에서만 다열 확장

## 컴포넌트 가이드

- 버튼: primary / outline
- 입력: 라벨 + 인풋 + 보조설명(optional)
- 카드: 제목 + 설명 + 본문
- 상태: success / error 메시지 박스

## 전달 포맷 (권장)

아래 항목을 포함해서 전달하면 구현 속도가 가장 빠름.

1. 화면별 캡처(모바일, 데스크탑)
2. 간격/타이포 수치(폰트, 라인하이트, 패딩)
3. 색상 토큰(HEX 또는 OKLCH)
4. 컴포넌트 상태(기본/로딩/에러/비활성)

## Replit 활용 방식

- 가능: UI 프로토타입 작성/실험
- 불가(직접 연동): 현재 레포와 자동 동기화

즉, Replit에서 시안을 만든 뒤 결과를 텍스트/스크린샷/코드로 전달하면,
이 프로젝트 코드베이스에 맞게 내가 정식 반영하는 방식이 가장 안전하다.
