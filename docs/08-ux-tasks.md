# UX Tasks (Club & Match)

## P0

### 1) 클럽명 변경 (클럽장 전용)
- 목표: 클럽장이 클럽명을 변경할 수 있게 한다.
- 범위:
  - 클럽 상세에 `클럽 설정` 섹션 추가
  - 클럽장(owner)에게만 수정 폼 노출
  - 변경 성공 시 즉시 화면 반영
- 수용 기준:
  - 비클럽장은 수정 버튼이 보이지 않거나 비활성화된다.
  - 변경 시 `audit_logs`에 `club.name.updated` 기록이 남는다.

### 2) 클럽명 중복 방지
- 목표: 동일한 클럽명으로 변경/생성되지 않도록 한다.
- 범위:
  - DB unique 인덱스(정규화: `lower(trim(name))`)
  - 서버/클라이언트 에러 매핑
- 수용 기준:
  - 중복 시 저장이 차단되고, 사용자 메시지가 노출된다.

### 3) 내 클럽 닉네임 변경
- 목표: 사용자가 클럽별 내 닉네임을 변경할 수 있게 한다.
- 범위:
  - 클럽 상세(또는 설정)에 `내 닉네임 변경` 폼
  - 변경 시 멤버 목록/경기 입력 라벨 즉시 반영
- 수용 기준:
  - 길이/공백 검증을 통과한 경우에만 저장된다.

### 4) 클럽 내 닉네임 중복 방지
- 목표: 같은 클럽에서 닉네임 충돌을 막는다.
- 범위:
  - `(club_id, lower(trim(nickname)))` unique 제약
  - 사용자 메시지 매핑
- 수용 기준:
  - 동일 닉네임 저장 시 실패하고 명확한 메시지를 반환한다.

## P1

### 5) 설정 UX 개선
- 목표: 설정 편집 과정을 실수 없이 진행하게 한다.
- 범위:
  - 입력 유효성 실시간 검증
  - 저장/취소 버튼 상태 제어
  - 성공/실패 피드백 일관화
- 수용 기준:
  - 무효 입력에서는 저장 불가 상태가 유지된다.

### 6) 권한 가시화
- 목표: 누가 어떤 설정을 바꿀 수 있는지 명확히 보여준다.
- 범위:
  - 설정 영역에 권한 라벨/설명 추가
  - 비권한자용 read-only UI
- 수용 기준:
  - 권한이 없는 사용자는 행동 가능한 CTA를 보지 않는다.

### 7) 예외 기록 처리 고도화
- 목표: 3게임 종료 같은 예외 기록을 혼동 없이 표시한다.
- 범위:
  - 히스토리/상세에서 `진행/중단` 상태 일관 표기
  - 필터(완료/예외) 추가
- 수용 기준:
  - 사용자가 예외 기록만 빠르게 구분할 수 있다.

## P2

### 8) 변경 이력 화면
- 목표: 클럽명/닉네임 변경 이력을 확인할 수 있게 한다.
- 범위:
  - `audit_logs` 기반 타임라인 UI
- 수용 기준:
  - 변경자/시간/변경 전후 값이 노출된다.

### 9) E2E 시나리오 테스트
- 목표: 권한/중복/예외 흐름 회귀를 방지한다.
- 범위:
  - owner 변경 성공
  - non-owner 변경 차단
  - 중복 차단
  - 예외 기록 표시
- 수용 기준:
  - CI에서 시나리오 테스트가 통과한다.

### 10) 용어 통일
- 목표: 도메인 용어 혼선을 제거한다.
- 범위:
  - `사이드/세트/라운드` -> `팀/게임` 중심으로 통일
  - 문구 사전 관리
- 수용 기준:
  - 주요 화면에서 혼재 용어가 사라진다.
