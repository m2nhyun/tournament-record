# Club Record Rules

## Product Direction

- `club record`는 단순 경기 기록이 아니라 클럽 운영용 데일리 매치 시스템이다.
- 핵심 운영 단위는 `데일리 매치 이벤트`다.
- 기존 `matches`, `match_schedules`를 억지로 재사용하지 않고 별도 도메인으로 설계한다.

## Auth / Roles

- 인증은 카카오 계정 기반
- 한 사용자는 여러 클럽 소속 가능
- 권한은 `관리자 > 운영진 > 회원 > 게스트`
- 관리자는 클럽당 1명
- 운영진은 여러 명 가능

## Ranking

- 랭킹은 숫자 순위만 사용
- `1위`가 최고
- 항상 연속 순번 유지
- 공동 순위 없음
- 수동 관리
- 랭킹은 관리자/운영진만 조회 가능
- 회원은 자기 랭킹도 볼 수 없음

## Group

- 랭킹과 그룹은 분리
- 그룹은 `A / B / C`
- 기본 비율은 `20 / 30 / 50`
- 그룹은 랭킹순으로 자동 배정
- 운영진이 비율 변경 가능
- 게스트는 랭킹은 없지만 그룹 지정 가능

## Daily Match

- 생성값:
  - 날짜
  - 시작시간
  - 종료시간
  - 코트 수
- 같은 날짜에 여러 이벤트 생성 가능
- 전부 복식
- 30분 슬롯 단위
- 참석자는 운영진/관리자가 관리
- 게스트 초대코드는 이벤트별 발급
- 게스트는 코드 진입 시 바로 참석 확정
- 게스트 데이터는 이후 회원 전환을 고려해 누적 보존
- 이벤트 상태는 참가자/경기 진행도에 따라 자동으로 갱신된다

## Participants

- 참석자 추가/삭제는 자동 편성 전후 모두 가능
- 참석자 변경 후 자동 재편성은 자동 실행하지 않음
- 운영진이 수동으로 `재편성` 실행
- 이벤트에는 `assignment_dirty` 플래그를 두고, UI에는 `인원 변경됨` 같은 경고를 이 값으로 노출
- 늦참은 30분 단위 입력
- 이벤트 종료 1시간 전까지 등록 가능한 방향으로 설계
- 이미 편성된 참가자를 삭제하려 하면 경고가 필요
- 해당 참가자에게 `confirmed` 경기가 하나라도 있으면 참가자 삭제는 막고, 경기 단위 조정을 유도
- `pending_result` 또는 `cancelled` 경기만 걸려 있으면 그 경기들을 자동 삭제하고 슬롯을 해제한 뒤 참가자를 삭제
- 참가자 row를 직접 삭제해 경기 선수 구성이 깨지지 않도록 삭제는 RPC 경계를 우선한다

## Match Assignment

- 자동 편성과 수동 편성을 모두 지원
- 수동으로 특정 슬롯에 커스텀 경기 생성 가능
- 자동 편성은 남은 슬롯/인원만 대상으로 실행
- 자동 편성이 한 번 성공하면 `last_assignment_run_at`을 남기고 `assignment_dirty`를 해제한다
- 자동 편성 규칙은 수동 편성에는 강제하지 않음
- 단, 수동 편성이라도 같은 시간대에 한 참가자가 두 슬롯에 동시에 들어가는 것은 허용하지 않음
- 이벤트 시간/코트 변경 시 기존 참가/편성 데이터 초기화
- 단, `confirmed` 경기가 하나라도 있는 이벤트는 시간/코트 변경을 차단한다
- 자동 편성은 빈 슬롯이 있으면 인원을 계속 재사용해 최대한 채운다
- `인당 최대 2경기` 규칙은 사용하지 않는다
- 같은 날 같은 사람과 같은 매치에 함께 엮이는 횟수는 기본적으로 2회 이하를 선호한다
- 같은 페어(같은 팀 조합)는 같은 날 반복하지 않는 것을 선호한다
- 단, 참가자가 부족해 위 선호 규칙을 지키면 뒤 시간대 경기를 만들 수 없는 경우에는 조합 반복보다 슬롯 채우기를 우선한다
- 같은 시간대의 여러 코트에는 한 사람이 동시에 중복 배정되지 않는다
- 운영진이 볼 때 각 시간대별 빈 코트와 실제 편성 가능한 슬롯이 명확하게 구분되어야 한다
- 운영 보드에서는 이벤트 전체 미배정 인원뿐 아니라, 각 시간대별 `open slot`과 `available participant`가 분리되어 보여야 한다

## Event Status

- `draft`: 참가자가 없는 상태
- `open`: 참가자는 있으나 시작 전 상태
- `in_progress`: 시작 이후이며 아직 진행 중인 상태
- `completed`: 시작 이후이고 `pending_result` 경기가 없으며 이벤트 종료 시각도 지난 상태
- `cancelled`: 이벤트가 취소되었거나 소프트 삭제된 상태

## Result Rules

- 결과는 참가자 4명 중 1명이 입력
- 게스트는 결과 입력 불가
- 회원만 본인 경기 결과 입력 가능
- 운영진/관리자는 모든 경기 결과 수정/재입력 가능
- 스코어는 자유 입력 문자열
- 스코어는 필수
- 스코어로 승/패/무 자동 계산
- `5-5` 같은 동점은 무승부
- 입력 기한은 `이벤트 종료 시각 + 24시간`
- 미입력 경기는 취소
- 미입력 취소 후에도 운영진/관리자는 다시 결과 입력 가능

## Attendance / History

- `attendance_count`와 `match_count`는 분리 저장
- 출석은 기록용 지표로 유지
- `attendance_count`는 삭제되지 않은 이벤트에 회원 참가자로 등록된 횟수
- `match_count`는 자동/수동 구분 없이 결과가 입력되어 `confirmed`된 경기 수
- 히스토리 공개 범위:
  - 본인 히스토리는 본인만
  - 운영진/관리자는 전체 히스토리 조회 가능
  - 클럽 전체에는 월간 공개 카드만 제공
- 월간 공개 카드는 `승 많은 순 -> 승률 -> 랭킹순`

## Delete Policy

- 이벤트는 소프트 삭제
- 참석자는 목록에서 실제 삭제
- 단, 이미 확정된 경기에 연결된 참가자는 직접 삭제하지 않고 경기 단위 조정으로 유도
- 자동 생성 경기는 삭제 가능
- 결과 무효화 기능은 아직 범위 밖
