# Match History UI Guidelines

이 문서는 경기 히스토리 화면의 큰틀(원칙)과 디테일(체크리스트)을 고정한다.

## 1) 큰틀 원칙

- 히스토리는 `빠른 스캔`이 목적이다.
- 기본 흐름은 `목록 확인 -> 경기 상세 진입`이며, 상세 정보는 상세 페이지에 위임한다.
- 같은 데이터라도 `카드`와 `리스트`의 역할을 분리한다.
  - 카드: 맥락(유형/상태/날짜/게임별 스코어) 확인
  - 리스트: 최소 정보(팀명 + 최종 스코어) 고속 확인
- 승/패 판정은 반드시 `현재 로그인 사용자 기준`으로 계산한다.
- 필터는 필요할 때만 여는 `접힘 기본값`을 유지한다.

## 2) 보기 모드 규칙

### 카드 모드

- 헤더 순서: `결과 배지(승/패/무) -> 단식/복식 -> 상태`.
- 날짜, 미완료 게임 포함 여부, 게임 스코어 요약을 노출한다.
- 점수는 `팀A x:y 팀B` 형식으로 중앙 강조한다.
- `submitted` 상태에서는 `기록됨` 라벨과 함께 확인 대기 의미가 드러나야 한다.
- `disputed` 상태에서는 재검토 필요 의미가 드러나야 한다.

### 리스트 모드

- 한 줄 구조만 사용한다: `팀A  x:y  팀B`.
- 아래 항목은 노출하지 않는다:
  - 아이콘
  - 승/패 뱃지 텍스트
  - 단식/복식
  - 기록됨/예외기록
  - 날짜
  - 게임 스코어 텍스트
- `confirmed` 경기만 승패 배경색을 사용한다.
  - 승: `green-light`
  - 패: `red-light`
  - 무: `gray-50`
- `submitted`, `disputed`는 중립 배경을 유지하고 상태는 상세/카드에서 해석한다.

## 3) 필터/탐색 규칙

- 필터 트리거는 Chevron 토글 버튼으로 제공한다.
- 기본은 닫힘(`collapsed`) 상태를 유지한다.
- 1차 필터는 `경기일(date)` + `상대 이름`만 제공한다.
- 필터 0건 결과 시에는 전용 EmptyState를 노출한다.

## 4) 무한 스크롤 규칙

- 초기 렌더는 제한된 개수만 노출한다(현재 16개).
- 하단 sentinel + IntersectionObserver로 점진 로딩한다.
- 필터 조건 또는 보기 모드가 바뀌면 visible window를 초기화한다.

## 5) 디자인 토큰 규칙

- 승/패 강조는 글로벌 토큰을 사용한다.
  - `--color-green-primary`, `--color-green-light`
  - `--color-red-primary`, `--color-red-light`
- 컴포넌트 내부 하드코딩 색상은 지양하고 토큰 우선 사용한다.

## 6) QA 체크리스트

- 본인이 패배한 경기에서 리스트 배경이 red 계열로 보이는가?
- 본인이 승리한 경기에서 리스트 배경이 green 계열로 보이는가?
- `submitted` 경기에서 리스트 배경이 중립으로 보이는가?
- `disputed` 경기에서 리스트 배경이 중립으로 보이는가?
- 리스트 모드에서 메타 정보(아이콘/날짜/상태)가 보이지 않는가?
- 카드 모드에서 경기 맥락(유형/상태/날짜/게임 스코어)이 유지되는가?
- 카드 모드에서 `submitted`는 `기록됨`, `disputed`는 `재검토 필요`로 해석되는가?
- 필터 패널이 기본 접힘이며, 토글로 열고 닫히는가?
- 필터 적용 후 무한스크롤이 초기화되어 첫 페이지부터 보이는가?
- 모바일(320px)에서 팀명/스코어가 깨지지 않는가?
